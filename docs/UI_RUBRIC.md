# UI Rubric

## Non-completion conditions

- UI improvement without visual, DOM-smoke, or screenshot evidence is incomplete.
- Text-density review without a concrete artifact is incomplete.
- More cards, labels, or notes do not solve the current dashboard problem by
  themselves.
- Aesthetic commentary without a next reversible UI action is incomplete.

## Evaluation axes

| Axis | Pass condition |
|---|---|
| Current state | First viewport states blocker/attention/access status plainly. |
| Priority ordering | One ordered priority path is visible before raw evidence. |
| Active decision | The selected review item has a clear decision question and next safe action. |
| Evidence adjacency | Source paths and relevant counts are close to the selected item. |
| Ignore path | Locked lanes and raw validation/smoke/action rows are secondary. |
| Card control | Independent cards are not the top-level layout grammar. |
| Japanese UX | User-facing review labels can be interpreted naturally by a Japanese operator. |
| Static boundary | No server, scheduler, writeback, or execution control is introduced. |
